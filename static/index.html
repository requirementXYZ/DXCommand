📡 DX COMMAND — QUICK START

What is this? DX Command is a dashboard for CW (Morse) and FT8 operators chasing rare DX and DXpeditions. It gathers DX-cluster spots, WSJT-X decodes, propagation data and beacon schedules in one place — and one click tunes your radio (via OmniRig, which supports most transceivers) to the right frequency, mode and split. It modernizes the classic VE3NEA / DX Atlas tools: Band Master, Morse Runner, Faros and friends.

First five minutes

  1. Try it with no radio: everything you see in demo mode is simulated — a Bouvet DXpedition pileup on CW and FT8. Click spots, watch the rig panel follow.
  2. Click a spot in the table or band map. The rig tunes, the mode is set, and if the comment says UP 2 the TX split is set too. The world map draws the path.
  3. Import your log (⬆ IMPORT ADIF, top right). Now the red/orange/yellow ★ flags mean you actually need that one: new DXCC / new band / new mode.
  4. Watch your target: click a DXpedition in the left panel (or add a call in WATCH LIST). You'll get an alert whenever it appears — on any mode.
  5. Go live: open ⚙ SETUP and switch the radio to OmniRig, spots to a real cluster, FT8 to WSJT-X. Changes apply instantly and are remembered.

Reading the colours

  • Red — all-time new DXCC entity (ATNO)
  • Orange — entity worked, but new on this band
  • Yellow — band worked, but new on this mode
  • Purple — watch-list callsign
  • CW spot  /  FT8/FT4 spot

Panels

Every panel has a small i icon — hover or click it for what that panel does and how to use it. The NCDXF beacon clock doubles as a live propagation check: click a beacon and listen. The CW TRAINER (top right) is a Morse Runner-style pileup simulator for practising before the real thing.

Demo vs live: run_demo.bat always starts fully simulated; run.bat starts with whatever you saved in ⚙ SETUP. Full docs in README.md.
